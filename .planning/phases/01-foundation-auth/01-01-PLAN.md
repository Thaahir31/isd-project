<task type="auto">
  <name>Task 1: Initialize Project Structure and Docker</name>
  <files>Dockerfile, docker-compose.yml, requirements.txt, src/main.py, src/config.py, .env.example</files>
  <action>
    Create a standard FastAPI project structure. 
    1. Define `requirements.txt` with fastapi, uvicorn, sqlalchemy, psycopg2-binary, pydantic-settings.
    2. Create `Dockerfile` (Python 3.11-slim) and `docker-compose.yml` with 'api' and 'db' (Postgres 15) services.
    3. Create `src/config.py` using Pydantic Settings to load DATABASE_URL from environment.
    4. Implement `src/main.py` with a simple health-check endpoint `GET /health`.
  </action>
  <verify>
    <automated>docker compose up -d --build && curl -f http://localhost:8000/health</automated>
  </verify>
  <done>Docker containers are running and the health-check endpoint returns {"status": "ok"}.</done>
</task>

<task type="auto">
  <name>Task 2: Configure SQLAlchemy and Alembic</name>
  <files>src/database.py, alembic.ini, src/models/base.py</files>
  <action>
    Setup database connectivity and migration framework.
    1. In `src/database.py`, initialize SQLAlchemy engine (sync) and SessionLocal.
    2. Create a base model in `src/models/base.py` using DeclarativeBase.
    3. Initialize Alembic with `alembic init alembic`.
    4. Configure `alembic/env.py` to use the application's Base.metadata and DATABASE_URL from config.
  </action>
  <verify>
    <automated>docker compose exec api alembic revision --autogenerate -m "initial" && docker compose exec api alembic upgrade head</automated>
  </verify>
  <done>Alembic is configured and the first migration successfully applies to the PostgreSQL container.</done>
</task>
